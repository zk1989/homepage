About

General
All design and layout ideas of this website are my own. I started working without any AI support. Then some of the more challenging parts and refactoring were dealt with GitHub Copilot in Visual Studio Code and through vibe-coding with Windsurf. Below I list just some of the things I've learned in the process.

Tools I used:
- I created the pattern in the background with: Pattern Monster
- I took CSS for thought bubble from here: Codepen
- at different stages of working on this, I used Visual Studio Code, JetBrains WebStorm and Windsurf

Lessons learned - AI:
- it's greedy - any imprecision or incorrectly set limitations can result in unasked for changes which are entirely out of scope
- creating things with AI may sound like a good idea, but the amount of time spent on verifying code correctness, understanding it, refactoring, removing unused and unasked for code kind of balances out the advantages...
- AI is doing well when asked for help in case of specific problems, debugging issues, some final code review; not necessarily laying out foundations for anything (yet)
- content that AI provides gives an impersonal vibe, it's generic - it's good if that's what you're looking for

Lessons learned - front-end:
- it's useful to fix scrollbar as fixed (always visible) - without this, the page will "jump" depending on the width of the content
- if default html/body values in terms of margins and padding are not overwritten, the browser defaults will be applied, called "user agent styles" or "user agent settings"
- header is a broader term - it can contain logo and navigation, while all "nav" elements are only for navigation purposes
- for composition debugging (flex/grid structures) it's good to ask AI to color all divs differently and draw their outlines
- I learned basic usage and definition of variables (custom properties) in CSS
